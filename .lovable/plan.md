

## خطة: أداة WILAY احترافية مستقلة

### المشكلة الحالية
أداة WILAY مدمجة داخل صفحة ModPackager الضخمة (2300+ سطر) مما يجعلها صعبة الوصول. الأداة الحالية تدعم فقط: عرض أساسي، تصدير PNG، واستبدال بسيط.

### الحل
إنشاء صفحة مستقلة `/wilay` بواجهة احترافية مع ميزات متقدمة.

---

### الخطوة 1: إنشاء صفحة مستقلة `src/pages/WilayViewer.tsx`

صفحة كاملة مستقلة تحتوي على:

**واجهة احترافية:**
- شريط أدوات علوي ثابت (toolbar) مع اسم الملف وأزرار التصدير
- شريط جانبي بقائمة الصور المصغرة (thumbnails) للتنقل السريع
- منطقة عرض رئيسية كبيرة للصورة المحددة
- لوحة معلومات تفصيلية

**ميزات احترافية جديدة:**
1. **تكبير/تصغير + سحب**: زوم بعجلة الماوس مع سحب الصورة (Pan & Zoom)
2. **خلفية شطرنجية**: لعرض الشفافية بوضوح (checkerboard pattern)
3. **مقارنة قبل/بعد**: عند استبدال صورة، عرض المقارنة جنباً لجنب
4. **عرض القنوات**: RGB / Alpha / Red / Green / Blue كل على حدة
5. **معلومات مفصلة**: الأبعاد، التنسيق، عدد الـ mipmaps، حجم البيانات، offset في الملف
6. **تصدير متعدد**: PNG فردي، تصدير الكل كـ ZIP، تصدير بصيغة أصلية (Raw Mibl)
7. **استبدال محسّن**: معاينة الصورة الجديدة قبل التأكيد مع تحذيرات الأبعاد
8. **بحث وفلترة**: فلترة حسب التنسيق (BC1/BC3/BC7/JPEG) أو الأبعاد
9. **عرض Hex**: عرض بايتات الـ footer الخام للصورة المحددة
10. **وضع الشبكة/القائمة**: التبديل بين عرض الصور كشبكة أو قائمة تفصيلية

### الخطوة 2: تحديث التوجيه في `src/App.tsx`

إضافة مسار `/wilay` لازى لودنج الصفحة الجديدة.

### الخطوة 3: تحديث الصفحة الرئيسية `src/pages/Xenoblade.tsx`

تحديث رابط زر WILAY ليوجه إلى `/wilay` بدلاً من `/mod-packager#wilay-viewer`.

### الخطوة 4: إزالة قسم WILAY من ModPackager

نقل الكود بالكامل للصفحة الجديدة وإزالته من ModPackager لتقليل حجم الملف.

---

### التفاصيل التقنية

- الصفحة تستخدم `wilay-parser.ts` الموجود دون تعديل
- التكبير/التصغير عبر CSS transform + wheel event
- خلفية الشطرنج عبر CSS gradient pattern
- عرض القنوات عبر Canvas pixel manipulation (فصل RGBA)
- تصدير ZIP عبر مكتبة JSZip الموجودة أو يدوي
- الشريط الجانبي يستخدم ScrollArea مع صور مصغرة قابلة للنقر
